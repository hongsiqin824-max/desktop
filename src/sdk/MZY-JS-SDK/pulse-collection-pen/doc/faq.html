浏览器与授权

为什么没有蓝牙选择窗口？

  • 确认使用支持 Web Bluetooth 的 Chrome 或 Edge。
  • 确认页面运行于 HTTPS 或 localhost。
  • requestDevice() 必须直接由用户点击事件触发。

能否不弹窗直接连接？

首次授权必须弹窗。之后可调用 getAuthorizedDevices() 获取已授权设备并使用 connectAuthorizedDevice() 连接。

连接与协议

isConnected 为 true，为什么命令仍失败？

GATT 连接成功不代表协议通道已经就绪。请以 canExecute 为准；出现 PROTOCOL_NOT_READY 时不要发送命令。

自动重连如何使用？

const client = PulseCollectionPenClient.create({
  autoReconnect: true,
  reconnectAttempts: 3,
  reconnectDelayMs: 1000,
});

达到最大次数仍失败会触发 reconnect-failed。

采集问题

采集前必须调用 startCollection() 吗？

不需要。连接成功后调用 setPulsePosition() 设置部位即可，设备上报稳定数据时 SDK 会自动收集。startCollection() 只用于主动清空上一轮缓存并重新设置部位。

使用层需要统计采样数量吗？

不需要。稳定采集规则由 SDK 内部管理，使用层只需关注采集进度、单档完成和整轮完成事件。

收到 collection-invalid 怎么办？

提示用户重新稳定按压并继续采集。结合事件中的 reason 判断是提前结束、重新变得不稳定、压力档错位还是协议帧无效。

收满后设备仍在发送同档数据怎么办？

SDK 会自动保留有效的稳定采集结果并忽略同档后续冗余数据，不会重复触发单档完成事件。

页面性能建议

  • 实时曲线先缓存数据，每约 50ms 批量刷新一次，建议只显示最近 100 个点。
  • 进度 UI 每约 100ms 最多更新一次。
  • 日志使用固定长度队列，例如最多保留最近 300 条。
  • 断开、清空或重连失败时取消页面待执行的刷新定时器。
页面节流只应用于 UI 渲染，不要丢弃 SDK 的稳定采集事件；完整结果以完成事件为准。
