浏览器问题

如果无法调用串口，请先检查：

  • 是否使用支持 Web Serial API 的浏览器。
  • 页面是否运行在 HTTPS 或 localhost。
  • 串口授权是否由用户手动触发。

连接问题

如果复现没有开始，请确认：

  • 已执行 requestPort()。
  • 已执行 open()。
  • 设备当前未被其他页面占用。
  • 调用 sendData() 前已准备正确格式的 dataList。

sendData 数据有什么要求

  • dataList 类型为 number[][]。
  • 数据通常为 9 组或 18 组。
  • coefficient 范围为 0.5 到 1.2，默认值为 0.8。
  • 如来自脉诊仪采集结果，可使用记录中的 motorData 字段。
使用采集结果复现时，请确保数据顺序按照 side、type、position、pretype 排序。

收到 left-hand-unsupported 怎么办

该事件表示当前设备版本不支持左手把脉。建议业务侧提示用户切换右手模式，或更换支持左手的设备版本。

client.on('left-hand-unsupported', () => {
  console.warn('当前设备版本不支持左手把脉');
});

什么时候调用 stop 和 close

  • stop() 用于停止当前复现。
  • close() 用于关闭串口连接。
  • 一般建议先停止当前复现，再关闭连接。
