注册方式

client
  .on('realtime-data', (data) => console.log(data))
  .on('progress', (progress) => console.log(progress))
  .on('send-completed', () => console.log('复现完成'))
  .on('left-hand-unsupported', (data) => console.warn(data))
  .on('log', (message) => console.log(message))
  .on('error-message', (message) => console.error(message));

事件列表

SDK 继承 EventTarget，统一通过 client.on('事件名', handler) 订阅事件。

事件名触发场景回调数据格式
realtime-data设备返回实时复现数据时触发。{ cad, gad, chad }
progress每组数据发送完成后触发。{ index, cmd, pres }
send-completed本轮数据发送完成后触发。void
left-hand-unsupported当前设备版本不支持左手把脉时触发。{ cad, gad, chad }
logSDK 输出日志时触发。string
error-message发生设备或 SDK 错误时触发。string

实时数据

对应 realtime-data，用于接收设备实时返回的寸、关、尺数据。

{
  cad: number;
  gad: number;
  chad: number;
}
字段说明
cad寸。
gad关。
chad尺。

发送进度

对应 progress，用于观察 sendData() 的发送进度。

{
  index: number;
  cmd: number;
  pres: number;
}
字段说明
index当前发送完成的数据组序号。
cmd发送指令值。
pres压力或力度相关值。

完成事件

对应 send-completed，本轮数据全部发送完成后触发。

client.on('send-completed', () => {
  console.log('复现完成');
});
如果业务侧需要恢复按钮状态或记录复现完成时间，建议在该事件中处理。

左手不支持

对应 left-hand-unsupported，当前选择左手把脉，但设备版本不支持左手把脉时触发，同时会触发 error-message。

client.on('left-hand-unsupported', () => {
  console.warn('当前设备版本不支持左手把脉');
});

错误与日志

对应 log 和 error-message，用于调试与异常处理。

事件用途
logSDK 日志输出。
error-message设备或 SDK 错误输出。

返回数据格式

string;
