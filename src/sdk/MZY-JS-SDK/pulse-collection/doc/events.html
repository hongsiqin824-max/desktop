注册方式

client
  .onMeasurement((data) => console.log(data))
  .onPulseRecord((record) => console.log(record))
  .onPulseGroup((group) => console.log(group))
  .onCollectionCompleted((result) => console.log(result))
  .onDeviceNo((deviceNo) => console.log(deviceNo))
  .onDeviceStatus((flag) => console.log(flag))
  .onStatusChange((status) => console.log(status))
  .onLog((message) => console.log(message))
  .onError((message) => console.error(message));

事件列表

下面的表格把事件名、对应方法、触发场景和回调数据格式放在一起，方便快速查找。

事件名对应方法触发场景回调数据格式
measurementonMeasurement实时测量值变化时触发。{ measurement: { cadval, gadval, chadval }, tagName }
pulse-recordonPulseRecord生成单条脉象记录时触发。{ side, tagName, position, positionName, type, pretype, pretypeName, pressure, rawAds, rawPulse, checkCount, timestamp }
pulse-grouponPulseGroup一组脉象记录整理完成时触发，通常包含寸、关、尺三条。{ records: PulseRecord[], tagName }
collection-completedonCollectionCompleted本轮采集完成后触发。{ mode, side, analysis, records, tagName, timestamp }
device-noonDeviceNo读取到设备编号时触发。string
device-statusonDeviceStatus设备运行状态变化时触发。boolean
status-changeonStatusChange设备状态节点变化时触发。{ tagName, message, pretype? }
logonLogSDK 输出日志时触发。string
error-messageonError发生设备或 SDK 错误时触发。string

实时数据

对应 onMeasurement，用于接收实时测量值。

{
  measurement: {
    cadval: number;
    gadval: number;
    chadval: number;
  };
  tagName: string;
}

单条脉象记录

对应 onPulseRecord，用于保存采集过程中的单条脉象记录。

{
  side: number;
  tagName: string;

  position: number;
  positionName: string;

  type: number;

  pretype: number;
  pretypeName: string;

  pressure: number;

  rawAds: number[];
  rawPulse: number[];

  checkCount: number;

  timestamp: string;
}
字段说明
side1=左手，2=右手。
position1=寸，2=关，3=尺。
pretype1=浮，3=中，5=沉。
rawAds原始脉象数据。
rawPulse原始压力数据。
checkCount数据长度。
timestampISO 时间字符串。

脉象分组

对应 onPulseGroup，通常是一组整理好的脉象记录，常见为寸、关、尺三条。

{
  records: PulseRecord[];
  tagName: string;
}
当业务需要阶段性展示或按组落库时，优先监听这个事件。

{

{

设备状态

对应 onDeviceStatus，返回设备当前运行状态。

true;
值说明
true设备运行中。
false等待接收指令。

状态变化

对应 onStatusChange，用于感知设备流程节点变化。

{
  tagName: string;
  message: string;

  pretype?: number;
}

常见状态包括：

  • device_start_pulse_search - 开始寻脉
  • device_stop_pulse_search - 结束寻脉
  • device_reset_complete - 复位完成
  • device_overall_pressure_start - 开始总按压力
  • device_overall_pressure_end - 结束总按压力
  • device_cun_pressure_start - 开始寸按压力
  • device_cun_pressure_end - 结束寸按压力
  • device_guan_pressure_start - 开始关按压力
  • device_guan_pressure_end - 结束关按压力
  • device_chi_pressure_start - 开始尺按压力
  • device_chi_pressure_end - 结束尺按压力
  • device_failure_pulse_search - 自动寻脉失败
当收到自动寻脉失败时，建议引导用户切换手动寻脉流程。

状态说明表

tagName说明
device_start_pulse_search设备开始进入寻脉流程。
device_stop_pulse_search设备结束寻脉流程。
device_reset_complete设备复位完成，可进入下一轮操作。
device_overall_pressure_start开始执行总按压力阶段。
device_overall_pressure_end结束总按压力阶段。
device_cun_pressure_start开始执行寸位按压力阶段。
device_cun_pressure_end结束寸位按压力阶段。
device_guan_pressure_start开始执行关位按压力阶段。
device_guan_pressure_end结束关位按压力阶段。
device_chi_pressure_start开始执行尺位按压力阶段。
device_chi_pressure_end结束尺位按压力阶段。
device_failure_pulse_search自动寻脉失败，设备进入等待处理状态。

pretype 说明

当状态为 *_pressure_start 时，会附带 pretype 字段。

pretype压力类型
1浮
3中
5沉

自动寻脉失败处理

当收到 device_failure_pulse_search 时，设备会进入约 50 秒等待状态。此时可以询问用户是否继续进入手动寻脉流程。

如果用户同意手动寻脉，调用：

await client.allowHandSearch();

如果用户不希望继续手动寻脉，可调用：

await client.rejectHandSearch();

该方法用于结束当前寻脉流程，避免设备继续等待人工介入。

在手动寻脉流程中，可继续通过以下接口控制探头：

client.setMoveUp();
client.setMoveDown();
client.setMoveForward();
client.setMoveBack();

调整完成后，可重新调用 start() 继续采集。
