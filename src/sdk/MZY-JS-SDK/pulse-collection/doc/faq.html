浏览器问题

如果无法调用串口，请先检查：

  • 是否使用支持 Web Serial API 的浏览器。
  • 页面是否运行在 HTTPS 或 localhost。
  • 串口授权是否由用户手动触发。

连接问题

如果采集没有开始，请确认：

  • 已执行 requestPort()。
  • 已执行 open()。
  • start(mode, side) 参数正确。
  • 设备当前未被其他页面占用。

自动寻脉失败怎么办

当收到 device_failure_pulse_search 时，设备会进入等待状态。建议的处理流程是：

  1. 提示用户切换到手动寻脉。
  2. 调用 allowHandSearch()。
  3. 使用方向控制接口调整探头。
  4. 调整完成后重新调用 start()。

什么时候调用 stop 和 close

  • stop() 用于结束当前采集流程。
  • close() 用于关闭串口连接。
  • 一般建议先停止采集，再关闭连接。
